Analysis of AI and IDE Configuration Folders
=============================================

Here is a detailed breakdown of what each of these hidden folders contains and what the code inside them does. 

All of these folders share a common theme: they contain configuration files, hooks, and instructions for various AI Coding Assistants and Integrated Development Environments (IDEs). Specifically, they are configuring a tool called "code-review-graph" which acts as an MCP (Model Context Protocol) server to give AI assistants advanced codebase analysis features.

---

### 1. .claude
- **What it contains**: `settings.json` and a `skills/` directory (containing files like `debug-issue/SKILL.md`, `refactor-safely/SKILL.md`, etc.).
- **What the code does**: 
  - `settings.json` contains command hooks ("PostToolUse", "SessionStart") that automatically update the `code-review-graph` whenever Claude modifies a file or starts a session.
  - The `skills/` files contain Markdown instructions teaching Claude how to properly use the code-review-graph tools (e.g., how to safely refactor or review changes efficiently).

### 2. .cursor
- **What it contains**: `mcp.json`.
- **What the code does**: Instructs the Cursor IDE on how to start the local `code-review-graph` Python MCP server so Cursor's AI can query your codebase's structure instead of just reading raw text.

### 3. .gemini
- **What it contains**: `settings.json`, `settings.json.bak`, `.sh` hook scripts, and a `skills/` directory.
- **What the code does**: This is exactly like the `.claude` folder but specifically tailored for Google's Gemini AI assistant. It provides the same hooks to update the graph and the same skill instructions for Gemini to follow when analyzing your code.

### 4. .github
- **What it contains**: `code-review-graph.instruction.md`.
- **What the code does**: This provides steering instructions (often read by GitHub Copilot) that tell the AI: "Always use the code-review-graph MCP tools BEFORE using traditional search tools." It dictates a specific workflow for exploring and reviewing code.

### 5. .kiro
- **What it contains**: `steering/code-review-graph.md`.
- **What the code does**: Similar to the `.github` instruction file, this gives specific system prompts to the Kiro AI assistant, teaching it when and how to use the code-review-graph tools over standard Grep/Glob file scanning.

### 6. .qoder
- **What it contains**: `mcp.json` and `settings.json`.
- **What the code does**: Configures the Qoder AI assistant. It combines the MCP server execution path (like Cursor) with the Post-action graph update hooks (like Claude/Gemini) so Qoder can sync and query your code graph.

### 7. .vscode
- **What it contains**: `mcp.json`.
- **What the code does**: Configures Visual Studio Code to start the local Model Context Protocol (MCP) server for the `code-review-graph`. This allows VS Code extensions (like Cline, Roo, or native VS Code AI features) to access the graph.
